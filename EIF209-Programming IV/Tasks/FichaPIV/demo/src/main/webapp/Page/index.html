Problema

Trata de un enfoque novedoso para permitir que las aplicaciones de JavaScript accedan a estructuras de datos C sin sobrecarga de rendimiento. Y esto se debe a que los lenguajes dinámicos como JavaScript no tienen un diseño de memoria fijo para los datos en tiempo de ejecución ni permiten accesos de memoria de bajo nivel, lo que dificulta la interoperabilidad con lenguajes como C.

Otras Solciones

Otras herramientas que se pueden utilizar para llegar a resultados similares son Google's V8 JavaScript engine. Pero el problema es que no almacenan datos en tiempo de ejecución a pesar de tener un rendimiento más alto, puede ser superado con este nuevo enfoque.

Experimento

La solución propuesta se enfoca en permitir que las aplicaciones JavaScript utilicen punteros C de forma transparente como receptores de accesos a objetos o accesos a matrices, por lo que el programador puede acceder a las estructuras de datos C como si fueran objetos JavaScript. Y para lograr esto utilizaron la herramienta Truffle modificándola tanto en su versión JavaScript y en C, generando diferentes operaciones de acceso en tiempo de ejecución dependiendo de si el receptor es un objeto JavaScript o un puntero C.
